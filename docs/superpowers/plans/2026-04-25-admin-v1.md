# Admin v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a protected `/admin` page where admins can inspect users/reservations, update user role/office, and cancel future reservations.

**Architecture:** Add a focused `admin-service` for DB reads/writes and permission-free business operations. Server pages/actions enforce current-user role and call the service. UI stays in one `/admin` page with local tab state.

**Tech Stack:** Next.js App Router, Prisma, React server/client components, Vitest.
